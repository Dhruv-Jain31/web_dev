creating a cluster : kind create cluster --name local
Delete the cluster : kind delete cluster -n local
Create the node setup : kind create cluster --config clusters.yml --name local
getting pods and nodes: kubectl get nodes & kubectl get pods
see the exact HTTP request that goes out to the API server: kubectl get nodes --v=8
Start a pod : kubectl run nginx --image=nginx --port=80
Check the logs : kubectl logs nginx
Delete a pod:  kubectl delete pod nginx
Applying the manifest : kubectl apply -f manifest.yml
Create a new namespace: kubectl create namespace backend-team
Get all the namespaces : kubectl get namespaces
Get all pods in the namespace: kubectl get pods -n my-namespace
Get the deployments in the namespace: kubectl get deployment -n backend-team
allows to directly edit the live deployment manifest: kubectl edit deployment test-app -n default
Set the default context to be the namespace: kubectl config set-context --current --namespace=backend-team
Get the configmap: kubectl describe configmap ecom-backend-config
Check the StorageClass details : kubectl get storageclass standard -o yaml
Get available storage classes: kubectl get storageclass
Check the CPU usage: kubectl top pods
See the hpa average usage: kubectl get hpa
